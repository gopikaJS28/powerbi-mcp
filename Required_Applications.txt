# Power BI MCP Integration – Complete Setup & Usage Guide

## 📌 Introduction

This guide provides step-by-step instructions to set up and run the **Power BI MCP (Model Context Protocol) Integration Project**. It is intended for developers who want to connect an AI agent to **Power BI Desktop** for querying, analyzing, and interacting with semantic models using DAX.

---

# 🎯 What This Project Does

* Connects to a running Power BI Desktop instance
* Extracts metadata (tables, columns, measures, relationships)
* Executes DAX queries programmatically
* Enables AI-driven interaction with Power BI models

---

# 🖥️ 1. System Requirements

| Requirement | Specification                    |
| ----------- | -------------------------------- |
| OS          | Windows 10 / 11                  |
| RAM         | Minimum 8 GB (16 GB recommended) |
| Storage     | 5–10 GB free space               |

---

# 🐍 2. Install Python

### ✅ Required Version

* Python **3.10**

### ⚠️ Important Notes

* Avoid Python 3.13 → incompatible with `pythonnet`
* Python 3.11+ may cause issues with .NET bindings

### 🔧 Installation Steps

1. Download Python 3.10
2. During installation:

   * ✔ Enable **“Add Python to PATH”**
3. Verify installation:

```bash
python --version
```

---

# 📁 3. Clone the Repository

Clone the project from GitHub:

```bash
git clone https://github.com/gopikaJS28/Power-BI-MCP.git
cd Power-BI-MCP
```

---

# 📦 4. Create Virtual Environment

```bash
python -m venv powerbi-env
```

Activate:

```bash
powerbi-env\Scripts\activate
```

---

# 📚 5. Install Required Python Packages

```bash
pip install pythonnet psutil mcp
```

### 📌 Package Description

| Package   | Purpose                                          |
| --------- | ------------------------------------------------ |
| pythonnet | Enables Python to use .NET libraries (ADOMD.NET) |
| psutil    | Detects Power BI Desktop processes               |
| mcp       | Runs the Model Context Protocol server           |

---

# 🧩 6. Install ADOMD.NET (Mandatory)

### 📌 What is this?

ADOMD.NET allows Python to connect to the Power BI semantic engine.

### 📍 Expected Path

```
C:\Program Files\Microsoft.NET\ADOMD.NET\160\
```

---

## 🔧 Installation Options

### Option 1 (Recommended)

Install:

* Microsoft Analysis Services ADOMD.NET

### Option 2

Install:

* SQL Server Feature Pack (includes ADOMD.NET)

---

## ✅ Verification

Ensure this file exists:

```
Microsoft.AnalysisServices.AdomdClient.dll
```

---

# 📊 7. Install Power BI Desktop

### ✅ Requirement

Install the latest version of Power BI Desktop.

### ⚠️ Important

* Open a `.pbix` file before running the MCP server
* This starts the local Analysis Services engine (`msmdsrv.exe`)

---

# ⚙️ 8. How the System Works

```
Power BI Desktop (.pbix)
        ↓
msmdsrv.exe (local engine)
        ↓
ADOMD.NET
        ↓
Python Connector
        ↓
MCP Server
        ↓
AI Client (Claude / others)
```

---

# 🚀 9. Running the MCP Server

From the project directory:

```bash
python src/server.py
```

---

# 🔗 10. MCP Configuration

Configure your MCP client:

```json
{
  "command": "C:/path/to/powerbi-env/Scripts/python.exe",
  "args": ["C:/path/to/Power-BI-MCP/src/server.py"]
}
```

# ⚠️ 12. Important Development Rules

## ❌ Avoid

* Using `print()` → breaks MCP JSON communication
* Using emojis (e.g., ✅ ❌) → causes encoding errors

## ✅ Use

```python
logger.info("message")
```

OR:

```python
print("message", file=sys.stderr)
```

---

# 🛠️ 13. Troubleshooting Guide

## ❌ ADOMD.NET not found

**Fix:**

* Install ADOMD.NET
* Verify correct path

---

## ❌ JSON Parse Error

**Cause:**

* Output printed to stdout

**Fix:**

* Remove all `print()` statements

---

## ❌ Unicode Error

**Cause:**

* Emojis in logs

**Fix:**

* Remove emojis

---

## ❌ No Power BI Instance Found

**Fix:**

* Open a `.pbix` file in Power BI Desktop

---

## ❌ Git Issues (Optional)

If pushing to GitHub fails:

* Configure Git user name and email
* Remove nested `.git` folders if present
